Why are DAT files not supported?

DAT (XML) files, used by tools like MAME and CLR MAME Pro, are a great tool to check, rebuild and convert romsets.

However, my goal is to have an easy-to-use software, and DAT files cannot be edited easily. Plus, they lack many useful informations: screen resolution, arcade hardware (Bally/Midway, CPS X, Sega System X...), emulation status, game type, etc.

Using a CSV file (wether exported from ArcadeItalia or completely custom-made) allows anyone with Excel (or any text editor) to filter the lists and create their own custom collection. You can even share them using Google Sheets.

You can convert a DAT file to a CSV file using this tool, if you already have a DAT file that you like to use.
